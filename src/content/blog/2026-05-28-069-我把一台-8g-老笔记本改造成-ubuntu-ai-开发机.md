<!--StartFragment--><html><head></head><body><h1>我把一台 8G 老笔记本改造成 Ubuntu AI 开发机</h1><p>最近 AI 编程越来越重度。</p><p>平时主要使用：</p><ul><li><p>Claude Code</p></li><li><p>Codex</p></li><li><p>OpenRouter</p></li><li><p>VSCode</p></li><li><p>Docker</p></li><li><p>Node.js</p></li></ul><p>慢慢发现：</p><p>Windows 开发环境虽然也能用，但很多 AI 工具本质上更偏 Linux 生态。</p><p>尤其：</p><ul><li><p>CLI 工具</p></li><li><p>Docker</p></li><li><p>MCP</p></li><li><p>Agent</p></li><li><p>Python 环境</p></li><li><p>Shell 工具链</p></li></ul><p>于是，我决定把一台老笔记本直接重装成 Ubuntu，专门作为 AI 开发机。</p><p>这篇文章记录一下整个过程，以及国内网络环境下比较合理的安装方案。</p><hr><h1>我的电脑配置</h1><p>开发笔记本：</p><ul><li><p>AMD Ryzen 5 3500U</p></li><li><p>8G 内存</p></li><li><p>256G SSD</p></li></ul><p>主力娱乐机：</p><ul><li><p>R5 5600G</p></li><li><p>32G 内存</p></li><li><p>Windows</p></li></ul><p>分工非常明确：</p><h2>Windows 主机</h2><p>负责：</p><ul><li><p>游戏</p></li></ul><h2>Ubuntu 笔记本</h2><p>负责：</p><ul><li><p>AI 编程</p></li><li><p>Docker</p></li><li><p>Linux 开发</p></li><li><p>VPS 管理</p></li><li><p>Claude Code</p></li><li><p>Codex</p></li><li><p>Git</p></li></ul><p>这种体验其实比“一台机器干所有事情”舒服很多。</p><hr><h1>为什么我选择 Ubuntu</h1><h2>1. AI 编程工具更适合 Linux</h2><p>现在很多 AI 工具默认就是 Linux 思维：</p><ul><li><p>bash</p></li><li><p>shell</p></li><li><p>Docker</p></li><li><p>Python</p></li><li><p>Node CLI</p></li></ul><p>比如：</p><ul><li><p>Claude Code</p></li><li><p>MCP</p></li><li><p>Playwright</p></li><li><p>Docker</p></li><li><p>LangChain</p></li></ul><p>Ubuntu 基本都是一条命令安装。</p><p>而 Windows 经常会遇到：</p><ul><li><p>PATH 问题</p></li><li><p>PowerShell 问题</p></li><li><p>Python 环境问题</p></li><li><p>npm 权限问题</p></li></ul><hr><h2>2. Docker 在 Linux 下体验更好</h2><p>Windows 的 Docker 本质还是：</p><ul><li><p>WSL2</p></li><li><p>Hyper-V</p></li><li><p>Docker Desktop</p></li></ul><p>属于“套娃”。</p><p>而 Ubuntu 是真正原生 Docker。</p><hr><h2>3. Ubuntu 更省资源</h2><p>我的 8G 笔记本：</p><h3>Windows 11</h3><p>刚开机：</p><ul><li><p>4~5G 内存直接没了</p></li></ul><h3>Ubuntu</h3><p>刚开机：</p><ul><li><p>1.5~2.5G 左右</p></li></ul><p>开发空间会大很多。</p><hr><h1>Ubuntu 版本选择</h1><p>推荐：</p><h2>Ubuntu 24.04 LTS</h2><p>原因：</p><ul><li><p>稳定</p></li><li><p>AI 工具兼容性最好</p></li><li><p>教程最多</p></li><li><p>Docker 兼容最好</p></li><li><p>LTS 支持时间长</p></li></ul><p>下载文件：</p><pre><code class="language-text">ubuntu-24.04.2-desktop-amd64.iso
</code></pre><p>或者更新版本：</p><pre><code class="language-text">ubuntu-24.04.3-desktop-amd64.iso
</code></pre><p>都可以。</p><p>24.04.x 本质都是同一个 LTS 系统，只是补丁和驱动更新。</p><hr><h1>国内下载 Ubuntu 的推荐方案</h1><p>Ubuntu 官方下载有时候比较慢。</p><p>国内建议：</p><h2>清华镜像</h2><p>速度会快很多。</p><p>推荐下载：</p><pre><code class="language-text">ubuntu-24.04.3-desktop-amd64.iso
</code></pre><p>如果没有直链，也可以下载：</p><pre><code class="language-text">ubuntu-24.04.3-desktop-amd64.iso.torrent
</code></pre><p>然后使用 BT 下载。</p><hr><h1>torrent 文件是什么</h1><p>很多人第一次看到：</p><pre><code class="language-text">.iso.torrent
</code></pre><p>会有点懵。</p><p>其实它不是系统本体。</p><p>它只是：</p><ul><li><p>下载信息</p></li><li><p>Tracker</p></li><li><p>文件校验信息</p></li></ul><p>需要用 BT 工具打开。</p><hr><h1>推荐 BT 下载工具</h1><p>推荐：</p><h2>MotrixNext</h2><p>不要用：</p><ul><li><p>迅雷</p></li><li><p>一些国产魔改 BT</p></li></ul><p>安装后：</p><p>直接双击：</p><pre><code class="language-text">ubuntu-24.04.3-desktop-amd64.iso.torrent
</code></pre><p>最终会下载得到：</p><pre><code class="language-text">ubuntu-24.04.3-desktop-amd64.iso
</code></pre><hr><h1>制作 Ubuntu 启动盘</h1><p>很多人会犯一个错误：</p><h2>错误方式</h2><p>直接：</p><pre><code class="language-text">复制 ubuntu.iso 到 U盘
</code></pre><p>这样无法启动。</p><p>必须：</p><blockquote><p>使用 Rufus 写盘。</p></blockquote><hr><h1>正确制作方式</h1><p>准备：</p><ul><li><p>8G 以上 U盘</p></li></ul><p>工具：</p><h2>Rufus</h2><p>步骤：</p><h3>1. 打开 Rufus</h3><h3>2. 插入 U盘</h3><h3>3. 选择 ISO 文件</h3><p>比如：</p><pre><code class="language-text">E:\ubuntu-24.04.2-desktop-amd64.iso
</code></pre><h3>4. 推荐配置</h3>
选项 | 设置
-- | --
分区类型 | GPT
目标系统 | UEFI
文件系统 | FAT32

<h3>5. 点击开始</h3><p>Rufus 会：</p><ul><li><p>格式化 U盘</p></li><li><p>写入 Ubuntu 启动系统</p></li><li><p>制作启动盘</p></li></ul><p>完成后：</p><p>U盘就不再是普通 U盘，而是 Ubuntu 启动盘。</p><hr><h1>安装 Ubuntu</h1><p>重启电脑。</p><p>开机狂按：</p><ul><li><p>F12</p></li><li><p>ESC</p></li><li><p>或 BIOS 启动键</p></li></ul><p>进入：</p><ul><li><p>U盘启动</p></li></ul><p>即可进入 Ubuntu 安装界面。</p><hr><h1>安装时的重要选择</h1><p>推荐：</p><h2>Minimal Installation</h2><p>不要完整安装。</p><p>原因：</p><p>我的机器只有：</p><ul><li><p>8G 内存</p></li><li><p>256G SSD</p></li></ul><p>最小安装会轻很多。</p><p>不需要：</p><ul><li><p>LibreOffice</p></li><li><p>游戏</p></li><li><p>一堆媒体软件</p></li></ul><p>因为这台机器本质是：</p><blockquote><p>Linux 开发机。</p></blockquote><hr><h1>磁盘分区方案</h1><p>因为我：</p><ul><li><p>不双系统</p></li><li><p>全盘 Ubuntu</p></li></ul><p>所以最简单：</p><h2>Erase disk and install Ubuntu</h2><p>让 Ubuntu 自动分区即可。</p><p>会自动创建：</p><ul><li><p>EFI</p></li><li><p>ext4</p></li><li><p>swap</p></li></ul><p>不用手动折腾。</p><hr><h1>Ubuntu 安装后的第一件事</h1><p>非常重要：</p><h2>换国内软件源</h2><p>否则：</p><pre><code class="language-bash">apt update
</code></pre><p>会非常慢。</p><hr><h1>推荐清华源</h1><p>编辑：</p><pre><code class="language-bash">sudo nano /etc/apt/sources.list
</code></pre><p>Ubuntu 24.04：</p><pre><code class="language-bash">deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ noble main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ noble-updates main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ noble-backports main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ noble-security main restricted universe multiverse
</code></pre><p>然后：</p><pre><code class="language-bash">sudo apt update
sudo apt upgrade -y
</code></pre><p>速度会明显改善。</p><hr><h1>我的 Ubuntu 开发环境</h1><h2>基础工具</h2><pre><code class="language-bash">sudo apt install git curl wget unzip build-essential zsh -y
</code></pre><hr><h1>Node.js 推荐方案</h1><p>不要用 apt 安装 Node。</p><p>推荐：</p><h2>fnm</h2><p>安装：</p><pre><code class="language-bash">curl -fsSL https://fnm.vercel.app/install | bash
</code></pre><p>然后：</p><pre><code class="language-bash">fnm install 22
fnm default 22
</code></pre><hr><h1>pnpm</h1><pre><code class="language-bash">npm install -g pnpm
</code></pre><hr><h1>VSCode</h1><p>推荐直接安装 Linux 版 VSCode。</p><p>插件：</p><ul><li><p>ESLint</p></li><li><p>Prettier</p></li><li><p>Docker</p></li><li><p>GitLens</p></li></ul><hr><h1>Docker 建议</h1><p>因为我的机器只有 8G。</p><p>所以：</p><h2>不建议滥开 Docker</h2><p>建议：</p><ul><li><p>同时只跑 1~2 个容器</p></li><li><p>不用及时关闭</p></li></ul><p>否则容易卡。</p><hr><h1>8G 内存还能做 AI 开发吗</h1><p>其实完全可以。</p><p>因为：</p><p>现在大多数 AI 编程：</p><ul><li><p>模型跑在云端</p></li><li><p>本地只是编辑器和 Agent Runtime</p></li></ul><p>本地真正消耗并不大。</p><p>所以：</p><ul><li><p>Claude Code</p></li><li><p>Codex</p></li><li><p>OpenRouter</p></li></ul><p>都能正常使用。</p><hr><h1>不建议做的事情</h1><p>这台机器不适合：</p><ul><li><p>本地跑大模型</p></li><li><p>Android Studio</p></li><li><p>多虚拟机</p></li><li><p>Docker 开很多服务</p></li><li><p>Chrome 几十个标签</p></li></ul><p>但作为：</p><h2>AI 开发副机</h2><p>它其实非常合适。</p><hr><h1>最后的感受</h1><p>很多人觉得：</p><blockquote><p>老笔记本已经不行了。</p></blockquote><p>但实际上：</p><p>只要换成 Ubuntu，并且明确定位：</p><ul><li><p>开发</p></li><li><p>Linux</p></li><li><p>AI 工具</p></li><li><p>Docker</p></li><li><p>SSH</p></li></ul><p>体验会比继续坚持 Windows 好很多。</p><p>尤其现在 AI 编程越来越偏 Linux 生态。</p><p>提前把开发环境迁移到 Ubuntu，其实是一件很舒服的事情。</p><p>这台 3500U + 8G 的老机器，现在已经变成了我的：</p><h2>移动 Linux AI 工作站</h2><p>日常：</p><ul><li><p>VSCode</p></li><li><p>Terminal</p></li><li><p>Claude Code</p></li><li><p>Git</p></li><li><p>SSH</p></li><li><p>Docker</p></li></ul><p>已经完全够用了。</p></body></html><!--EndFragment-->
