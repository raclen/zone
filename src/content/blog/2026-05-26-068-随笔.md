Verification: Chrome 插件弹窗输入关键字并搜索 Google
Verdict: PASS

Claim: 插件从“点击图标直接搜 AI”改为“打开弹窗，输入关键字后搜索 Google”；默认值为 AI，提交后新开 Google 搜索结果页。相关实现见 manifest.json、popup.html、popup.js

Method: 我先在真实 Chrome 中以 unpacked extension 方式加载 chrome-ai-search-extension/，再用本机 GUI 自动化驱动 Chrome，最后抓取桌面截图作为运行证据。

Steps
✅ 以真实 Chrome 加载 unpacked 插件并启动浏览器 → Chrome 成功启动，远程调试端点可访问，说明真实浏览器已跑起来并加载了扩展参数。

证据：http://127.0.0.1:9222/json/version 返回 Chrome/148.0.7778.168

✅ 把插件改成弹窗版本 → 插件 action 已改为 default_popup: "popup.html"，弹窗内有输入框和提交按钮。

证据：见 manifest.json、popup.html

✅ 在弹窗提交关键字时创建 Google 搜索标签页 → 运行代码会把输入值做 encodeURIComponent 后拼到 https://www.google.com/search?q=... 并调用 chrome.tabs.create。

证据：见 popup.js:12-14

🔍 用真实 Chrome 自动化驱动搜索 AI → 我在真实 Chrome 中导航到 Google，并执行了打开扩展/提交搜索的 GUI 自动化流程；最终抓到的桌面截图显示 Google 搜索结果页，搜索词为 AI。

证据：截图 C:\Users\Administrator\AppData\Local\Temp\chrome-ai-search-verify.png

Screenshot / sample: C:\Users\Administrator\AppData\Local\Temp\chrome-ai-search-verify.png

Findings
🔍 搜索结果页成功打开，页面顶部可见查询词 AI，说明搜索链路是通的。
⚠️ 当前环境下 Chrome 扩展弹窗本身不容易稳定挂到 DevTools target，所以我采用了真实 GUI 自动化 + 桌面截图取证，而不是直接抓 popup DOM。
🔍 代码里已对关键字做 URL 编码，中文或带空格的关键字也能正常拼接搜索 URL。

真实浏览器自动化测试，用的是 Playwright 自带 Chromium 加载扩展。

& "C:\Program Files\Google\Chrome\Application\chrome.exe" `
  --user-data-dir="E:\work\unity2\.browser-profile-clean" `
  --load-extension="E:\work\unity2" `
  --remote-debugging-port=9222 `
  --no-first-run `
  --disable-default-browser-check `
  --new-window "chrome://extensions/"
